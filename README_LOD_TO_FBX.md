# LOD to FBX Converter

This script combines multiple Level of Detail (LOD) OBJ files generated by The Grove forest system into single FBX files suitable for game engines like Unity and Unreal Engine.

## Quick Start

```bash
# Convert all LOD files to FBX format
python src/growpy/combine_lods_to_fbx.py --input_dir data/output --output_dir fbx_output

# Check what files would be processed
python src/growpy/combine_lods_to_fbx.py --input_dir data/output --check_only

# Apply scaling during export (e.g., for Unity which prefers smaller units)
python src/growpy/combine_lods_to_fbx.py --input_dir data/output --scale 0.01 --output_dir unity_assets
```

## Requirements

- **Blender with Python API**: Install with `pip install bpy`
- **Alternative**: Use Blender's built-in Python: `/path/to/blender --background --python combine_lods_to_fbx.py`

## What It Does

1. **Groups LOD files by species** (e.g., all Beech LODs together)
2. **Imports each LOD level** into Blender as separate objects
3. **Names objects properly** for game engine LOD systems (e.g., `Species_LOD0_Ultra`)
4. **Exports as FBX** with optimized settings for game engines
5. **Creates manifest files** documenting what was exported

## LOD Levels

The script processes 6 LOD levels per species:

- **LOD0_Ultra**: Highest detail for hero objects
- **LOD1_High**: High detail for primary gameplay
- **LOD2_Medium**: Medium detail for standard viewing
- **LOD3_Low**: Low detail for background objects  
- **LOD4_VeryLow**: Very low detail for distant objects
- **LOD5_Minimal**: Minimal detail for far background

## Output

For each species, you get:

- `Species_LODs.fbx` - FBX file containing all LOD levels as separate objects
- `lod_manifest.txt` - Report of what was exported

## Game Engine Integration

### Unity

1. Import the FBX files into your Assets folder
2. Select the FBX file and configure LOD Group:
   - LOD0: 100% - 60%
   - LOD1: 60% - 30%
   - LOD2: 30% - 15%
   - LOD3: 15% - 5%
   - LOD4: 5% - 1%
   - LOD5: 1% - 0%

### Unreal Engine

1. Import FBX files to Content Browser
2. Open Static Mesh editor
3. Configure LOD levels in the LOD settings
4. Set appropriate screen sizes for each LOD

## Example Workflow

1. Generate LOD files using Grove:

   ```bash
   python run_demo.py  # Your Grove script
   ```

2. Convert to FBX:

   ```bash
   python src/growpy/combine_lods_to_fbx.py --input_dir data/output --output_dir game_assets
   ```

3. Import into your game engine and configure LOD groups

## Troubleshooting

**"Blender not available"**

- Install Blender Python API: `pip install bpy`
- Or run with Blender: `/Applications/Blender.app/Contents/MacOS/Blender --background --python combine_lods_to_fbx.py`

**"No LOD files found"**

- Check files follow naming pattern: `Species_LOD_Level.obj`
- Use `--check_only` to see what files are detected

**Large file sizes**

- Use `--scale 0.01` for Unity (converts from meters to centimeters)
- Consider using only LOD2+ for runtime (LOD0-1 for heroes only)

## Command Line Options

```
--input_dir     Directory containing LOD OBJ files (default: data/bu/output)
--output_dir    Directory to save FBX files (default: fbx_output)  
--scale         Scale factor to apply (default: 1.0)
--check_only    Only check for LOD files, don't process them
```
