Skeleton

Also see: Dynamics, Bones
The Grove Skeleton node creates a bone for every internode in the tree, and adds attributes that are great for simulations. The skeleton allows you to deform and animate trees. To build an optimized skeleton for more basic animation, use the new Bones node instead.


The node builds the bones as poly line primitives. The connected option defaults to on, and this means that the first node of each branch shares the same point as the parent branch node.


Attributes

  • gr_branch_id on the primitive level, links the primitive to the branch index.
  • mass on the point level, the total mass of the continuation of the branch, including side branches.
  • pscale on the point level, the radius of the node in meters.

Note that the mass and pscale attributes don’t use the gr_ prefix, that is because they are names that are expected by some of Houdini’s built-in nodes.


Weights

To link each point of the tree geometry to the correct bone, we have to add bone weight attributes. Houdini has a strange way of handling these and it is not possible to set them entirely in Python, so you will have to add two nodes after building your tree.


The build node creates an attribute gr_skeleton_joint_id, which is an integer number that points to the correct bone. We will convert this to bone capture data, as Houdini likes to call it.


Add a Point Wrangle with the following code…

export int @boneCapture_index[];
i[]@boneCapture_index = array(i@gr_skeleton_joint_id);

export float @boneCapture_data[];
f[]@boneCapture_data = array(1.0);

Feed the output into a Capture Attribute Pack node and you’re ready to deform your tree with a Joint Deform node.


Manual animation

You can pose the KineFX skeleton with a Rig Pose node.


Wind breeze animation

This is a work in progress.
To quickly add a basic wind sway to the skeleton, use a Rig Attribute Wrangle with the following code. This turbulence effect uses the radius of each branch node (the pscale attribute) to control the flexibility and the frequency of the sway.


float amount = chf("amount") * 0.002;
float noisy = chf("noisy") * 0.05;
float noise_scale = chf("noise_scale") * 0.1;

// Radius controls flexibility.
float flex = 1.0 / @pscale * 0.01;
flex = pow(flex, 0.8);

float angle = 0.0;
float height = @P.y;

// @Time * flex * 0.5 increases the frequency on thinner branches.

angle = sin(@Time*3) * amount;
angle += (-0.4 + noise((@P * noise_scale) + {1,0,0} * @Time * flex * 0.5)) * noisy;
angle *= flex;
angle *= height;

rotate(4@localtransform, angle, {1,0,0});
rotate(4@localtransform, angle, {0,0,1});

Polywire preview

If you don’t want to set up the weights and deformation, but instead just want a quick preview of the skeleton, you could use the PolyWire or PolyWire UV node from Houdini labs. This won’t get you all the attributes and it also won’t give you the deformed twig duplication geometry.


Using the pscale attribute (radius) as the Scale Attrib.


An Attribute Wrangle node can set the resolution attribute with @resolution = @pscale*80;. Use this resolution attribute as the PolyWire’s Div Attrib.
