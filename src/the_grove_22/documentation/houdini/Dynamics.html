Dynamics

Also see: Skeleton


The Skeleton node converts the simulation into a wireframe skeleton that can either be turned into a KineFX skeleton, or simulated directly with a Vellum Solver. The Skeleton node calculates the mass and thickness at every point in the tree, which Vellum can use to create a physics simulation.



Pin

After building the skeleton, pin the base points of the tree to keep them grounded. You can also do this in the Vellum Constraints node, but this way it is more obvious.


Use group patterns to pin points. For example 0-6 or @P.y<1.


Vellum Hair

Thickness and mass are both essential for realistic physics. The Grove calculates these for realistic bending while the tree grows, and now the Skelton node adds both attributes to be used in further dynamics simulations.


In the Geometry tab of the Vellum Constraints node, set Mass to Unchanged, to use the already available mass attribute. Do the same for Thickness, but this time Houdini expects an attribute called pscale, which is actually a radius rather than a thickness. The Grove Skeleton node creates the pscale attribute for each point in the skeleton.


Important! No matter the bend and stretch stiffness you set in the Vellum Hair constraints, the tree is going to be an elastic, wobbly mess unless you drastically increase the number of simulation steps. This can be done in two ways, either with more constraint iterations, or with more sub steps.


Let’s try increasing the constraint iterations up to around 500, up from the default of 100. The tree looks rigid and moves in a natural way, but you will see that points can sometimes enter the collision mesh and stick to it - they will be dragged on by the collision mesh and stretch the tree unnaturally. This happens because the collision object is moving too fast for the simulation to keep up.


Now let’s try the second way, with more sub steps, 30 perhaps, depending on how fast your collision object is moving. Great!


To sum it up, creating a non-flexible, stiff structure is not easy for Vellum, requires lots sub steps, and takes time to bake.
