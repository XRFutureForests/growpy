# PVE Python API: Direct Preset Authoring (Experimental)

UE 5.7 exposes the Procedural Vegetation Editor (PVE) plugin via the Python
API. This opens a complementary path to the existing USD skeletal-mesh
assembly export: instead of (or in addition to) shipping pre-grown geometry,
growpy can hand its **PVE JSON recipes** to PVE itself, which re-grows the
tree inside UE using its own simulation.

**Status:** experimental scaffolding only — the production path remains the
USD skeletal-mesh assembly export. Both pipelines can run from the same
growpy export and are not mutually exclusive.

## The two paths, side by side

| Aspect | USD Assembly path (production) | PVE preset path (new) |
|---|---|---|
| What growpy ships | Skeletal mesh USDA + twigs + textures | PVE JSON recipe |
| Geometry source | Grove simulation in growpy | PVE simulation inside UE |
| Calibration | growpy yield-table calibration baked in | PVE re-runs growth from JSON params |
| Twig placement | growpy assembly bound to skeleton | PVE Foliage Distributor + Foliage Palette |
| Wind | Dynamic Wind via skeletal mesh | Native PVE skeletal output |
| Edit in UE | No (re-export from growpy) | Yes (PVE Graph nodes: Carve, Gravity, Bone Reduction, ...) |
| MegaPlants compatibility | Compatible structure | Native PVE consumer |
| LOD/Nanite | Nanite Assembly via USD schemas | Native PVE Nanite output node |

The two paths are complementary, not redundant. Use the USD path for trees
that must match field measurements exactly (yield-table calibration, biomass
studies). Use the PVE path when you want artists to tweak species in-engine
or when you want PVE Graph processing on top of growpy parameters.

## Relevant UE 5.7 Python classes

- [`unreal.ProceduralVegetationFactory`](https://dev.epicgames.com/documentation/en-us/unreal-engine/python-api/class/ProceduralVegetationFactory?application_version=5.7)
  — DataAsset factory. Subclass of `Factory`. Creates new
  `ProceduralVegetationPreset` assets in the Content Browser.
- [`unreal.ProceduralVegetationPreset`](https://dev.epicgames.com/documentation/en-us/unreal-engine/python-api/class/ProceduralVegetationPreset?application_version=5.7)
  — DataAsset. Holds the JSON recipe path plus output folders. Key properties:
  - `json_directory_path` (DirectoryPath) — points at the folder of growpy
    PVE JSONs
  - `plant_profile_name` (str) — which JSON to use as the active recipe
  - `foliage_folder` / `materials_folder` (DirectoryPath) — where PVE writes
    its outputs
  - `override_folder_paths` (bool) — must be true to use the explicit folders
  - `create_profile_data_asset` (bool) — auto-creates the per-profile data
    asset alongside the preset
  - `trunk_material_name` (str) — optional trunk material override
  - `preset_variations` (Array, **read-only**) — derived after build
- [`unreal.ProceduralVegetationGraph`](https://dev.epicgames.com/documentation/en-us/unreal-engine/python-api/class/ProceduralVegetationGraph?application_version=5.7)
  — extends `PCGGraph`. The actual node graph (Preset Loader, Carve, Gravity,
  Foliage Distributor, Mesh Builder, Output, ...). Exposes `nodes`,
  `input_node`, `output_node`, `user_parameters`, `generation_radii`. Could
  in principle be authored programmatically; for now we only create the
  preset DataAsset and let users wire the graph manually.

## What growpy already produces

The current pipeline already emits PVE JSON recipes — see
[pve_grove_mapper.map_grove_to_pve](../src/growpy/io/unreal/pve_grove_mapper.py)
and [pve_schema.py](../src/growpy/io/unreal/pve_schema.py). Files land in
`data/assets/pve_configs/<species>_pve.json` with the Quixel Megaplants
schema (globalAttributes, points, primitives). The JSONs are species
overrides that calibrate PVE growth simulation against growpy/yield-table
data.

What was missing: an automated way to **import those JSONs as
ProceduralVegetationPreset DataAssets** inside UE without clicking through
the editor.

## New: PVE preset import script generator

[pve_import_script.py](../src/growpy/io/unreal/pve_import_script.py)
generates a UE Python script that:

1. Walks the forest root recursively at runtime for
   `*_stems_unreal_pve.json` recipes (so adding species or trees doesn't
   require regenerating the script)
2. For each recipe calls
   `AssetTools.create_asset(..., factory=ProceduralVegetationFactory())`,
   naming the asset `PVE_<basename>` with the suffix stripped
3. Sets `json_directory_path` (to the recipe's own parent dir),
   `plant_profile_name`, `foliage_folder`, `materials_folder`,
   `override_folder_paths = True`, `create_profile_data_asset = True`
4. Saves each asset under a configurable Content Browser package path,
   skipping any that already exist (idempotent re-runs)

The script is auto-generated by the dataset and forest pipelines, gated
on the `unreal.generate_pve_presets` config flag (default: true). Output
lands at `data/output/forest/unreal_scripts/growpy_pve_preset_import.py`
alongside `unreal_import_trees.py` and `unreal_cleanup_trees.py`.

### Usage

```python
from pathlib import Path
from growpy.io.unreal import generate_pve_preset_import_script

script_path = generate_pve_preset_import_script(
    output_dir=Path("data/output/forest/unreal_scripts"),
    forest_root=Path("data/output/forest"),
    package_path="/Game/Assets/TheGrove/PVE",
    foliage_folder="/Game/Assets/TheGrove/PVE/Foliage",
    materials_folder="/Game/Assets/TheGrove/PVE/Materials",
)
```

Then in Unreal: right-click the generated `.py` file in the Content Browser
or in the file picker → **Execute Python File in Unreal**. The two import
flows (skeletal mesh assemblies + PVE presets) coexist in the same UE
project as parallel UObjects.

### Requirements

- UE 5.7+ with the **Procedural Vegetation Editor** plugin enabled
- The growpy PVE JSON folder must be readable from UE's filesystem
- Output Content Browser folders must be writable

### Limitations of current scaffolding

- Does **not** author the `ProceduralVegetationGraph` (the node graph that
  consumes the preset). Users still need to set up a graph manually or load
  a saved one — same as the MegaPlants/Quixel workflow.
- Uses `set_editor_property` with `DirectoryPath` structs; the exact struct
  shape may vary across UE 5.7 patches. Tested API expectations are based on
  the public Python API reference — if `DirectoryPath()` constructor is
  unavailable, the script logs a warning and continues.
- Skips the per-asset variation step (`preset_variations` is read-only;
  variations are derived when PVE builds the preset).

## Future directions

1. **Author the PVE Graph** programmatically. The `nodes` array on
   `ProceduralVegetationGraph` is exposed; we could prebuild a graph that
   wires Preset Loader → Foliage Distributor → Mesh Builder → Output for
   each species and save it as a UAsset alongside the preset.
2. **Round-trip wind data**. The Dynamic Wind asset action can also be
   driven from Python — the existing growpy wind JSON could be applied
   directly via `unreal.AssetTools` after PVE completes its skeletal mesh
   build.
3. **Bone reduction targets** from yield-table data. PVE's Bone Reduction
   node accepts target counts; growpy already knows the target bone budgets
   per distance class (see [unreal-vegetation.md](unreal-vegetation.md)
   notes on bone-count targeting).
